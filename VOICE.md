# Binky — voice & copy reference

Use this for **in-app strings** and **marketing/site** language. Same dry-warm Binky register everywhere; different surfaces, same spine.

## Core metaphor

A binky is a pacifier. It soothes fussy things. Your Downloads folder is the fussy thing. **Binky acts. The inbox calms.**

Write from three words: **calm, quiet, handled.**

## Who’s reading

- **Anyone non-technical** — no hashes, regex, or OCR in UI copy. Reassuring beats precise.
- **Students** — dry humor welcome; setup should feel optional.
- **People working** — receipts, contracts, exports. Binky should feel **smart and predictable**, not random.

Default: **short, confident, calm.** No corporate tone. No chirpy exclamation points.

## Tone

- **Dry and warm** — deadpan, not cold.
- **One-liner bias** — if it needs two sentences, cut one.
- **Binky is the actor** — “Binky filed it,” not “Your file was processed.”
- **Past tense for outcomes** — sorted, filed, handled.
- **Don’t explain the pacifier** — trust the metaphor.

## Patterns that work

- Inbox-as-toddler: “Fussy inbox. Meet Binky.” / “Files were screaming. Binky helped.”
- Imperative: “Give it to Binky.” / “Sh. Binky’s handling it.”
- Outcomes: “Sorted. Routed. Binky’d.”
- Calm framing: “Quiets the mess right down.” / “Your inbox, quietly handled.”

## In-app: chips & short labels

| Meaning | Copy |
| --- | --- |
| Duplicate skipped | `Already got one.` |
| Auto-filed receipt | `Looks like a receipt.` |
| Rule matched (named) | `Sent by “[rule]”.` |
| Content-style match (generic) | `Matched by content.` |
| From download source | `From [site].` |

## In-app: empty states

- `Nothing in Review. Binky’s on top of it.`
- `Nothing old enough to touch yet.` (aging preview)
- `All quiet in Downloads.`
- `No rules yet. Binky’s winging it.`

## In-app: session / stats lines

Plain words, no jargon:

- `12 sorted · 3 already had · 2 receipts filed · 1 retired`

Use **already had** (not “duplicates”), **receipts filed**, **retired** (not “archived” / “aged out”) where it fits.

## In-app: confirmations

State **what happens**, then label the button with the consequence.

- Body: `This clears Binky’s memory of files it’s seen. Duplicates won’t be recognized until they’re sorted again.`
- Buttons: `Forget everything` / `Keep it`

## In-app: natural-language rule flow

- Header: `Here’s what Binky heard.`
- Confirm: `Looks right. Save it.` (applies to the form; user still saves the rule in the editor if needed)
- Reject: `Not quite. Try again.`

## Marketing / site

**Headlines** — short declarations:

- `Fussy inbox. Meet Binky.`
- `The pacifier for your Downloads.`
- `Sorted. Routed. Binky’d.`

**Features** — benefit first, hide implementation:

- “Binky watches your Downloads and puts things where they belong.”
- “Knows a receipt when it sees one.”
- “Remembers what you already have.”
- “Cleans out old stuff on a schedule.”

## Words to avoid

| Avoid | Prefer |
| --- | --- |
| Streamline / optimize / workflow | cut or plain verb |
| Duplicate / hash / SHA (in UI) | already had, already got one |
| Archive / age out (user-facing) | retired |
| OCR / content match (user-facing) | looked at it, matched by content |
| Process / processed | sorted, filed, handled |
