How to install Binky

Pick the path that fits you — both put Binky in Applications.

macOS will say Binky can't be opened — that's Gatekeeper, not a virus. Both steps take less than a minute.

  1. Open the installer from your Downloads, then drag Binky into your Applications folder. Download didn't start? Download again.

  2. Open Terminal and paste this command:

    xattr -dr com.apple.quarantine /Applications/Binky.app

Requires Homebrew. In Terminal, run once to add this repo as a tap, then install the app:

brew tap heyderekj/binky https://github.com/heyderekj/binky
brew install --cask binky

Upgrading Binky: in Terminal, run brew update && brew upgrade --cask binky when you want a newer release. If macOS blocks the first launch, switch to the Download & install tab and run the Terminal command in step 2.

Easy updating — once you're set up, the app can check GitHub Releases for updates.

macOS blocked Binky

Because Binky isn't from the App Store, macOS quarantines it on first run. Paste this into Terminal to remove the flag:

xattr -dr com.apple.quarantine /Applications/Binky.app

People who popped in a Binky

Real quotes from GitHub Discussions. Want to share your take? Leave a review.

    macOS says Binky can't be opened — what do I do?

    That's Gatekeeper.

    Do I need to be technical to use this?

    Nope. Out of the box, automations watch a folder — Downloads still polite default — and sort calm buckets. Reach for Describe a rule once you crave specifics.

    Is it actually free?

    Open source on GitHub under MIT. Free.

    Does Binky upload my files?

    No. Sorting stays on your Mac. No accounts baked in.

    Mac only? Which macOS?

    Yes. SwiftUI, native frameworks, no cross-platform runtime. Requires macOS 14 Sonoma or later. On macOS 26 Tahoe, the UI uses Liquid Glass where available.

    Morning digest?

    Optional UserNotifications rollup when you crave tallies instead of babysitting batches. Default stays quiet.

    Where do files land after sorting?

    Each automation declares its sorted buckets and downstream actions — rename tokens, archive extraction, Finder-only DMG installs, Finder tag spreads, Trash when warranted.

    What file types go in which destination?

    Images (jpg, png, gif, webp, svg, heic…), PDFs, Media (mp4, mov, mp3, m4a…), Documents (doc, txt, md, xls…), Archives (zip, rar, 7z, tar…), Apps (dmg, pkg, app), Screenshots (matched by name pattern), Misc (everything else). Unknowns or ambiguous types go to Review.

    What about watching Downloads or synced folders?

    Binky watches the folder you pick (Downloads by default). For Dropbox, Drive, iCloud, etc., it reacts after the sync client finishes laying bytes on disk — same idea as catching a fussy saver.

    Can I set custom routing rules?

    Yep — names, extensions, kinds, sizes, sheltered Finder tags, download origins, OCR hints — then reroute, rename, tag, unpack archives, install DMGs quietly, trash when you swear it's junk.

    Does it move originals or make copies?

    Moves originals. Nothing duplicated, nothing left behind in Downloads. That's why the stable-file gate and the Review folder exist — Binky won't touch a file it isn't sure about.

    What if Binky isn't running when files land?

    The watch folder only acts while Binky is running. Files that arrived while it was off sit untouched — run Sort Now when you're ready and it'll sweep them up.

    Who made this and why?

    Derek Castelli builds client sites in Figma and Webflow. He wanted a Downloads workflow that behaved without acting sneaky.
